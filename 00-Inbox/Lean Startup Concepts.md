
1. Innovation Accounting A way to measure progress by proving that new features actually improve user behavior. It helps avoid wasting time on things that don't matter.

2. Learning Milestones Specific events that prove a business assumption (Leap of Faith) is true, such as a user successfully using a new feature for the first time.

3. ![[The Vanity metrics and The Availibility Bias]] Numbers like "Total Signups" that look good on paper but don't tell you if the product is actually successful.

4. Actionable Metrics Data that tells you exactly what to do next. For example, tracking if users who use a specific feature are more likely to return to the app.
Examples

• Innovation Accounting: Comparing user retention before and after launching a new feature to see its real impact.

• Learning Milestones: Validating if users actually find value in organizing chat history.

• Vanity Metrics: Looking at "Total Registered Users" which doesn't prove the app is being used.

• Actionable Metrics: Tracking what percentage of users who try a feature once come back to use it again.
