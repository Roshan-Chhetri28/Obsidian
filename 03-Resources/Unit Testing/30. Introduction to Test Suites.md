## Introduction to Test Suites
- If a file contains more than one unit then the test file will contain a lot of passing and faling test so it would not be clear which test belong to which unit therefore we use Test Suits.
	- The following is the step to Create Test Suits
		-  we import describe from vitest 
		- call describe()
			- it takes two argument 
				- string (used to describe which unit the suite belongs to)
				- callback (add all the test belonging to the unit here)
		- The execution and evaluation is still the same but is more organised
	Example:
```js
import {it, expect, describe} from 'vitest'

describe('the name/description of the unit you are triying to test', ()=>{
	 all the test functions 
})
```
